Intergrated Development Environments (IDEs)

License

Licensing is taken into consideration when picking an IDE. If this was a real world project the budgetting and the availability/support of the IDE may factor into what one is needed. Most of the open-source IDEs allow for intergration of developed pluggins adding features that are appealing for what is needed, whereas licensed IDEs such as Webstorm have intergrated alot of features already so there may not be a need for pluggins.

Operating System

Most IDEs can be used on Windows, Linux and OS-X (Mac) which are the known desktop OS by most people, although there are others. Other OS such as Android for a selection of smartphones also have support by certain IDEs. But for this project the main point is selecting an IDE that runs on Windows, Linux and OS-X as development will mainly be focused on these OS (with future endevours of porting to a smart device)

Features

As stated before most of the open-source IDEs have pluggins available to incorporate as needed. Taking this into consideration if there is a feature you want then there is most likely a pluggin for it (if not then there is an opening for a pluggin to be developed). With Webstorm it is a licensed program so must be purchased BUT has many features intergrated so there may not be a need to get pluggins. This could be put into the time scheme of things, although pluggins can easliy be searched for most IDEs it may take time to intergrate the features you need for the project. Whereas if the features are already present then this saves time intergrating and can focus on learning how to use the program (if there was no previous experience).

Open-source IDE

As the open-source IDEs can have pluggins and relatively the same features it is good to get a few reviews from those that have used it themselves. Searching for comparisons of IDEs results in many articles and reviews of a variaty of IDEs. Many of them break it down to bulletpoints, naming features, usability and the general look of the user interface of each IDE. One such article by Craig Buckler compared the IDEs; Atom, Brackets, Light Table and Sublime Text. To quote the article he suggested to follow this criteria:

  1. Suited to web development
  2. Cross-platform and work on Windows, Mac and Linux. Your preferences and settings should be available regardless of which OS youâ€™re using.
  3. General-purpose text editors which support typical web languages. You should be able to use the same application for HTML, CSS, JavaScript, PHP, Ruby, SQL, markdown and more.
  4. Highly customizable with cross-platform plugins and themes.
  5. Fast and stable. Launching should never be a once-a-day dread like some IDEs.
  6. Immediately usable without a steep learning curve or having to remember numerous keyboard shortcuts.

In using this criteria Craig explained the strengths and weaknesses of each IDE and squared them off against each other, giving each catogory a winner.

  1. The Interface: Brackets
  2. Syntax Support: Sublime Text
  3. Ease of Use and Learning Curve: Brackets
  4. Speed and Stability: Sublime Text
  5. Native Feature Set: Sublime Text
  6. Web Specific Features: Brackets
  7. PLugins and Extensions: Sublime Text
  8. Customization and Hackability: Atom
  9. The Future: Atom

Although Sublime Text toke most of the categories, Brackets was a close runner up in a selection of this catergories.

Along side individual reviews there are dedicated websites which provide comparisons to help with selecting the correct IDE.

Webstorm

As Webstorm is a licensed program and is available through Otago University resources it can be up for consideration, but if this were a real world project it may be a factor of "do we really need the Webstorm features?" as commercial use requires a license (if there is an intention to publish this application). Due to this Webstorm may not be needed for this project and an open-source IDE could surfice with selecting features that are required to complete the project.

What is the best choice?

Brackets would be a good IDE to use, due to it being open-source (developers create pluggins for it) and is maintained very well, available on all platforms (Windows, Mac, Linux) the learning curve (taking into account those within the group that have not had much experience with development) and the ability to incorporate features that are needed to help complete the project. But due to lack of TypeScript support (version is 2 years out of date) the next best option is Sublime Text, as it was highly rated in reviews and has many of the features that are needed for the project, much like Brackets (along side pluggins).
